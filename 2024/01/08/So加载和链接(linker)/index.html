JAVA层调用SO

Android 应用程序通过 System.loadLibrary() 或 Runtime.loadLibrary() 加载本地库。这些本地库通常是 .so 文件，它们包含了通过 JNI 与 Java 代码交互的本地方法。Android 的类加载机制和虚拟机（ART）会负责解析这些库，并将其加载到内存中。

LoadLibrary()

http://androidxref.com/9.0.0_r3/xref/libcore/ojluni/src/main/java/java/lang/System.java#1651

在 Java 中，Runtime 是一个代表当前 Java 应用程序运行时环境的类。它提供了与 Java 虚拟机（JVM）进行交互的多种功能，如内存管理、运行时参数管理、执行系统命令等。

**loadLibrary0**：

  • 这是 Runtime 类的一个本地方法，实际上是通过 JNI (Java Native Interface) 调用底层操作系统的功能来加载共享库。在 Runtime.getRuntime().loadLibrary0() 中，loadLibrary0 负责执行具体的库加载工作，通常由 JVM 实现和操作系统平台提供支持。

**VMStack.getCallingClassLoader()**：

  • 这个方法返回调用当前方法的类加载器（ClassLoader）。类加载器负责加载类和资源，而 VMStack.getCallingClassLoader() 是获取调用当前方法的类加载器。该加载器通常是调用 loadLibrary 方法的类的 ClassLoader。在 Android 中，VMStack.getCallingClassLoader() 通过获取调用当前方法的类加载器来确定在当前上下文中加载共享库的路径。Android 中的类加载器负责加载 Java 类，并且在加载库时，会根据不同的 ClassLoader 来决定如何查找本地库（包括从 lib 目录、apk 文件中等）。

步骤 描述
1. 检查库是否已加载 函数首先检查是否已经加载该库。如果库已加载，并且类加载器相同，则返回成功；如果类加载器不同，生成错误信息并返回 false。
2. 解码类加载器 解码传入的类加载器对象，并检查是否为 BootClassLoader。如果是，设置为 nullptr，否则获取类加载器的分配器。
3. 打开共享库 调用 android::OpenNativeLibrary 使用 dlopen 加载共享库。如果加载失败，返回 false，并在 error_msg 中记录错误。
4. 创建共享库对象 创建 SharedLibrary 对象表示已加载的库，并通过加锁保证线程安全，将库添加到全局库列表中。
5. 调用 JNI_OnLoad 检查库中是否包含 JNI_OnLoad 函数，如果有，调用它进行初始化。如果返回错误版本，记录错误信息；否则标记为成功。
6. 返回加载结果 设置共享库的加载结果并返回是否成功。

**__builtin_return_address(0)**：

  • 这是一个 GCC 提供的内建函数，用来获取当前函数的调用者的返回地址。
  • 传入的参数 0 表示获取当前函数的调用者的返回地址。如果传入 1，则表示获取当前调用者的调用者的返回地址。
  • 该返回地址可以用来了解调用栈的位置，通常用于调试或记录调用路径。

**__loader_dlopen**：

  • 这是 dlopen 的底层实现，负责加载共享库。通过传递 caller_addr，它可能会记录加载请求的调用源，或者用于进一步的调试/分析。
  • __loader_dlopen 可能是一个与操作系统或具体实现相关的自定义函数，通常用于封装 dlopen 的实际底层操作。

**dlopen**：

  • dlopen 是动态库加载的标准函数，用于加载共享库（.so 文件）到当前进程的内存空间中。
  • filename 是共享库的路径，flag 用于指定加载的方式（例如：是否立即绑定符号，是否允许卸载等）。
